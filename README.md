# StockQuantAPP - 个人量化策略移动端 V3.0 技术文档

## 1. 业务背景与定位

`StockQuantAPP` 是 `MyStockQuant` 个人量化分析系统的官方移动端承载形态。其核心使命是：**让量化监控装进口袋，实现随时随地的策略跟踪与复盘。**

在整体架构中，移动端定位为纯粹的**视图层 (View)**。它不负责沉重的全市场数据拉取和高并发的策略计算（这些由 Python 后端和定时任务完成）。移动端专注于以最优的交互体验，展示后端引擎计算好的四大股票池（试盘、回踩、突破、废弃）、高精度历史 K 线以及系统流转日志。

## 2. 技术栈与架构设计

- **前端框架**：UniApp (基于 Vue 3 语法)。
- **路由与配置**：原生 `pages.json` 配置，底部 TabBar 导航。
- **图表渲染引擎 (双模架构)**：
  - **Hybrid 混合模式**：基础版本采用 `<web-view>` 组件内嵌后端的 `static/app_kline.html`，利用 Web 端 ECharts 5.5.0 进行图表渲染。
  - **RenderJS 极速模式**：进阶优化使用 `renderjs` 在逻辑层与视图层建立高速通道，直接在 App 底层操作 ECharts 引擎，抛弃 WebView 承载，实现**原生级渲染性能**。
- **横竖屏适配策略**：
  - **App 级真横屏**：调用 `plus.screen.lockOrientation('landscape-primary')` 实现硬件级物理旋转，提供沉浸式看盘体验。
  - **H5 级伪横屏**：在非 App 环境下，依赖 CSS `transform: rotate(90deg)` 实现视觉层面的横屏兼容。
- **自适应监听**：集成 `ResizeObserver` 动态监听容器尺寸变化，确保图表在隐藏状态栏或切换视图时 100% 撑满屏幕。

## 3. 核心功能与页面逻辑深度解析

App 包含 3 个核心页面和 1 个侧边栏底座组件。

### 3.1 首页：策略雷达池 (`pages/index/index.vue`)

这是 App 的入口，负责展示量化引擎筛选出的最新标的。

- **UI 布局**：顶部为四个状态 Tab（👀试盘、📉回踩、🔥突破、🗑️废弃），下方为长列表滚动区。
- **数据流转逻辑**：
    1.  **初始化/下拉刷新**：触发 `fetchCounts()` 获取四个池子的最新数量，更新 Tab 标题角标。
    2.  **列表渲染**：触发 `fetchStockList(currentStatus)` 获取当前选中池的股票列表。
    3.  **交互跳转**：点击任意一只股票，携带 `code`、`name` 和 `status` 参数，路由跳转至 K 线详情页。
    4.  **生命周期联动**：利用 `onShow` 生命周期，确保每次从详情页返回或唤醒 App 时，列表数据自动刷新。
- **进阶交互功能**：
    - **手动排序**：支持长按“⋮⋮”手柄进行列表位置自由拖拽交换（需 888 授权码验证）。
    - **一键置顶**：点击“📌”图标将关注标的移至列表首位。
    - **涨幅排序**：点击表头“涨幅”实现降序/升序/默认的三态切换循环。
    - **数据预加载**：列表加载完成后，后台以每 300ms 一只的速度串行静默请求 K 线数据 (`silentPreloadKLines`)，保护内网穿透服务不被高频封锁。

### 3.2 深度复盘：K线全景图 (`pages/kline/kline.vue`)

这是全端技术含量最高、交互最复杂的页面。

- **渲染模式选择**：
    - **WebView 桥接模式**：页面仅包含一个铺满全屏的 `<web-view>` 组件，`src` 动态指向后端 `/static/app_kline.html`。
    - **RenderJS 原生模式**：在 App 环境下启用，利用 `renderjs` 直接在视图层操作 ECharts，实现毫秒级响应和**断网秒开**（配合缓存）。
- **终极横屏适配逻辑**：
    - **App 环境 (5+ App)**：`onLoad` 时强制锁定物理横屏，`onUnload` 时恢复竖屏。
    - **H5 / 小程序环境**：依赖 Web 侧内部的伪横屏 CSS 类进行视觉旋转。
- **沉浸式状态栏处理**：动态获取设备平台信息，精确计算安全区（刘海屏）边距，确保 K 线十字光标和顶部 LCD 数据条不被遮挡。
- **图表交互优化**：
    - **默认视图**：进入页面默认展示最近 **60 个交易日**，提供宏观趋势视野。
    - **手势增强**：DataZoom 滑块加高至 **32px**，手柄放大至 **150%**，完美适配手指触控操作。
    - **LCD 横幅**：实时显示当前光标所在日期的开/收/高/低、涨幅、换手率、成交额等核心指标。

### 3.3 审计与回溯：系统日志 (`pages/log/log.vue`)

为投资者提供上帝视角的审计追踪。

- **UI 布局**：按日期分组 (Group By Date) 呈现的时光轴列表。
- **功能特性**：
    - **聚合展示**：调用 `/api/logs` 获取历史操作记录（包含机器自动筛选、突破晋级、跌破淘汰、人工强制干预）。
    - **一键折叠/展开**：提供全局的“🔓 展开全部 / 🔒 收起全部”按钮，通过修改数据源 `expanded` 属性配合 `v-show` 极速切换视图状态。
- **业务意义**：了解引擎在何时何地因为什么逻辑买入或淘汰标的，是建立量化交易信任体系的关键一环。

## 4. 后端接口交互 (API 通信全景)

App 端通过 `uni.request` 与后端 FastAPI 服务通信。为了极速响应，App 调用的是专门定制的 **BFF 层接口**。

| 接口地址 | 请求方式 | 作用与逻辑说明 |
| :--- | :--- | :--- |
| `/api/counts` | GET | **获取全局角标**：返回 `{1: 数量, 2: 数量, 3: 数量, 99: 数量}`。App 固定传递 `show_special=false`，自动过滤 ST、科创板等不感兴趣板块。 |
| `/api/pool/app/{status}` | GET | **获取列表明细 (BFF层)**：后端在返回数据前已执行联合查询，直接返回计算好的 `latest_price`, `latest_change`, `turnover` 和格式化后的 `volume`，前端直接双向绑定渲染，无需额外计算。 |
| `/api/logs` | GET | **拉取审计日志**：返回按日期倒序排列的字典数据。 |
| `/api/run_strategy` | POST | **远程触发同步**：触发后端执行增量同步任务（需授权码）。 |
| `/static/app_kline.html` | GET | **请求静态图表容器**：图表内部会自行调用 `/api/kline/{code}` 获取数据并渲染。 |

## 5. 数据缓存与弱网优化

为了在移动网络环境（尤其是通过 Ngrok 内网穿透）下提供流畅体验，App 实现了深度缓存策略。

- **硬盘级持久化缓存**：使用 `uni.setStorageSync` 将 K 线 JSON 字符串直接写入手机存储。第二次打开同一只股票时，实现 **0 毫秒秒开**，甚至支持**离线复盘**。
- **静默预加载**：首页列表渲染完毕后，后台启动队列任务，串行预请求列表中前 N 只股票的 K 线数据，有效规避高频请求导致的服务器限流。
- **状态同步锁**：当 Web 端或后端定时任务正在执行时，App 端按钮会自动置灰并显示“执行中”，防止并发冲突导致数据库死锁。

## 6. 核心字段含义清单

在 App 首页列表中，绑定的字段具有如下业务含义：

| 字段 | 业务含义 |
| :--- | :--- |
| `code` / `name` | 股票 6 位代码与名称。 |
| `price` (`latest_price`) | 该标的在最近一个交易日的收盘价。 |
| `change` (`latest_change`) | 较前一交易日的涨跌幅（%）。用于控制文字颜色（红涨绿跌）及背景块颜色。 |
| `amount` | 当日成交金额。发生“倍量试盘”时该数值会显著放大。 |
| `turnover` | 当日换手率。用于判断主力资金的活跃程度。 |
| **标签 (Badge)** | `ST` (风险警示股)、`科` (科创板)、`创` (创业板)。 |

## 7. 安全风控与授权机制

为确保移动端操作不会误触发或破坏策略状态，系统内置了多层防护：

- **888 授权码**：所有涉及数据库修改或高危操作的请求（如增量同步、置顶排序、状态变更）均需在 Header 中携带有效的授权码 `X-Auth-Code`。
- **二次确认弹窗**：对“增量同步”和“深度重置”等高危操作，强制执行 `uni.showModal` 交互确认。
- **服务器配置管理**：侧边栏支持自定义 Ngrok 域名或内网 IP 地址，方便在不同网络环境下切换后端服务。

## 8. 项目运行与编译指南

### 8.1 环境准备
- 安装 **HBuilderX** (推荐使用正式版)。
- 确保 Python 后端 (`MyStockQuant`) 正在运行，且手机/模拟器与电脑处于同一局域网，或后端已通过 Ngrok 等工具暴露至公网。

### 8.2 配置文件修改
1. 打开 `pages/index/index.vue`、`pages/log/log.vue` 等涉及请求的页面。
2. 找到 `uni.request` 的 `url` 配置。
3. **重要**：将 `http://192.168.0.103:8000` 或示例域名修改为您真实的 Python 后端服务地址。

### 8.3 运行方式
- **H5 预览**：在 HBuilderX 中点击 `运行` -> `运行到浏览器`。此时横屏为 CSS 伪横屏。
- **真机调试**：使用 USB 数据线连接手机，点击 `运行` -> `运行到手机或模拟器`。**此模式下，物理横屏 (`plus.screen.lockOrientation`) 和 RenderJS 原生性能才会真实生效。**
